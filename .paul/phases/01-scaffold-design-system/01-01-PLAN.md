<objective>
## Goal
Ship a working v0.1 MVP in one pass: paste one or many public GitHub repo URLs into the dashboard, get back a fair, transparent, rubric-scored verdict per repo as an animated flashcard, expandable into a full detail view with graphs and a downloadable report. No auth.

## Purpose
This is the whole MVP loop, not a thin skeleton — GitHub ingestion, AI judging, and UI are tightly coupled (the UI shape depends on the verdict schema, the verdict schema depends on what ingestion can supply), so building them as one phase avoids rework.

## Output
- `lib/github.js` — repo URL parsing + REST ingestion (metadata, tree, README, manifests, commit activity)
- `lib/openrouter.js` — free-model discovery (live from `/models`, not hardcoded) + fallback rotation
- `lib/rubric.js` — 7-category rubric (sums to 100), prompt builder, JSON verdict parser/repair, server-side score verification
- `app/api/analyze/route.js` — POST `{ repoUrls: string[] }`, bounded concurrency, per-repo error isolation
- `components/dashboard/*` — RepoInputForm, VerdictCard (flashcard), VerdictDetail (modal, radar + bar charts), Leaderboard, loading/empty states
- `app/page.js` — the dashboard itself, is the app's only route in v1
- Dark "courtroom" theme tokens in `app/globals.css`
</objective>
